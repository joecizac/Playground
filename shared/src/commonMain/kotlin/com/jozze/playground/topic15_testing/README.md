# Topic 15: Testing

## Summary
Unit, integration, and UI testing strategies for Kotlin and Android.

## Subtopic Index
- Testing Pyramid
- Unit Testing
- Test Doubles
- Coroutine Testing
- Flow Testing
- UI Testing
- TDD

## Mental Model
Tests should make behavior cheap to verify and regressions easy to localize.

## Common Mistakes
- Testing implementation details instead of behavior.
- Using real dispatchers in coroutine tests.
- Making UI tests cover logic that belongs in unit tests.
