# Topic 13: Design Patterns

## Summary
Reusable approaches for object creation, structure, and behavior.

## Subtopic Index
- Singleton
- Builder
- Factory Method
- Adapter
- Decorator
- Facade
- Observer
- Strategy
- Command

## Mental Model
Patterns are vocabulary for tradeoffs, not templates to force everywhere.

## Common Mistakes
- Using a singleton where DI would make lifecycle and testing clearer.
- Adding pattern classes before duplication or complexity exists.
- Hiding simple logic behind unnecessary abstraction.
