# Topic 21: Compose Multiplatform Specifics

## Summary
Sharing UI with Compose across platforms and integrating platform-specific UI when needed.

## Subtopic Index
- Shared UI vs Shared Logic
- UIKitView
- AndroidView
- Compose Resources
- Navigation
- Lifecycle Mapping

## Mental Model
Share UI when the experience and platform constraints allow it; share logic when UI needs to stay native.

## Common Mistakes
- Assuming every platform should share identical UI.
- Ignoring lifecycle differences.
- Hardcoding resources instead of using Compose resources.
