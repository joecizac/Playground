# Topic 11: Multi-Modular Architecture

## Summary
Using Gradle modules to enforce boundaries, improve build behavior, and organize ownership.

## Subtopic Index
- Why Modularize
- Module Types
- Dependency Graphs
- Gradle Management
- Cross-Module Navigation

## Mental Model
Modules should encode architectural boundaries, not just folders.

## Common Mistakes
- Creating circular dependencies.
- Adding modules without a clear boundary.
- Sharing implementation classes across feature modules.
